# ADR-0025: Потік B — Data Quality & Activation Summary

- **Статус**: Implemented
- **Дата**: 2026-02-28
- **Автор**: Patch Master (GitHub Copilot)
- **Initiative**: potik_b_data_quality

## Контекст і проблема

Планувалась поетапна активація мульти-символьної роботи (Потік B) через config.json:symbols, з перевіркою якості даних, календарів, derived rebuild та верифікацією integrity. Після повного audit/rebuild виявлено:

- Дані для всіх 13 символів (XAU, XAG, FX, CFD) присутні, 3–5 місяців M1, всі TF
- 0 дублікатів по всіх символах і TF
- Календарі для всіх груп вже налаштовані
- Архітектура мульти-символьна, всі воркери ітерують symbols з config.json
- Derived rebuild працює, але більшість derived свічок побиті (data integrity порушена)

## Рішення

- Етап B (масова активація) пропущено — працюємо лише з XAU/USD
- Всі результати audit/rebuild зафіксовані тут для історії та наступних рішень

## Наслідки

- Всі символи готові до активації з точки зору архітектури та календарів
- Data integrity для derived TF потребує окремого audit/fix перед майбутньою активацією
- Потік C (SMC engine) стартує лише на XAU/USD

## Rollback

- Якщо потрібно повернутись до мульти-символьної активації: повторити data quality audit, rebuild, та перевірити integrity

## Notes

- Всі ключові висновки зафіксовані для уникнення повторного audit
- Потік B закритий, фокус на XAU/USD та SMC engine
