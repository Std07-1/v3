## Quality Axes

- **Ambition target**: **R1** — уніфікація наявних розрізнених hint-механізмів під один перемикач; нової поверхні немає.
- **Maturity impact**: **M3.5 → M4** — «підказки новачка» стають одним узгодженим governance-каналом (передбачувано, з одного місця); прибрано важкий 3-станний polite-таймінг тулбару. Знижень немає.

## Контекст

Підказки на чарті були **розкидані** по трьох незалежних механізмах:

1. **Тулбар-лейбли інструментів малювання** — власний «ввічливий» 3-станний таймінг ADR-0077 (показати двічі на hover, тоді замовкнути назавжди, скинути через 4с). Owner: непередбачувано — «я потім забуду», коли/чому лейбл з'являється.
2. **Hover-пояснення SMC-overlay** — зони (FVG/OB: страйк/статус/опис) і **всі** структури (BOS/CHoCH, свінги HH/HL/LL/LH, фрактали, displacement, inducement) — DOM-tooltip у `OverlayRenderer` (`_hitAreas` + `_tooltipEl`), **завжди активний**, незалежно ні від чого.
3. **Title-підказки хрому** — `title={$hintsOn ? … : undefined}` у CommandRail/ChartHud/ReplayBar/StatusBar — вже під спільним стором `uiHints` (☰-меню «Підказки»).

Owner-вимога: **усі підказки новачка = один перемикач**, щоб трейдер мав чистий графік і його нічого не відволікало; згруповані в одному місці, вмикаються/вимикаються спільно. Overlay-пояснення зон і структур він «довго думав чи варто» — вирішив: віднести до підказок новачка, щоб можна було **сховати повністю**.

---

## Рішення

### D1. Один стор — `uiHints` (`hintsOn`), перемикач у ☰-меню
Існуючий `stores/uiHints.ts` (localStorage `v4_show_hints`, **default OFF**, перемикач «Підказки» у `CommandRailOverflow`) стає **єдиним** governance-каналом для ВСІХ підказок новачка. Нічого нового — розширюємо охоплення.

### D2. Тулбар-лейбли — 2 стани (суперсідить ADR-0077 polite-таймінг)
`DrawingToolbar` імпортує `hintsOn` напряму (узгоджено з CommandRail/ChartHud). **Off** → повна тиша; **On** → лейбл на hover після `TIP_DELAY` (450 мс), тримається доки курсор на іконці. 3-станний polite-таймінг (`revealCounts`/`dwell`/`reset`, ~90 LOC) **прибрано** — два передбачувані стани замість трьох. Prop `alwaysShowHints` видалено; `$effect` ховає лейбл миттєво, якщо перемикач вимкнули поки лейбл видно.

### D3. Overlay-пояснення зон+структур — під той самий gate
`OverlayRenderer.setHintsEnabled(on)` (ставиться з ChartPane `$effect` на `$hintsOn`). `mousemove`-хендлер підказок має guard: **off → тиша** (і миттєво ховає відкриту підказку). Один механізм уже покривав і зони, і структури (спільний `_hitAreas`/`_tooltipEl`) — тож один gate вимикає все разом. Tooltip-елемент отримав клас `.smc-tooltip` (для верифікації/стилю).

### D4. Клас підказок = «новачок», не «функція»
Семантика: ці підказки навчають читати графік (що таке OB, чому BOS). Досвідчений трейдер їх вимикає (default) → чистий графік (Clean Chart Doctrine). Це **не** керуючі елементи — жодної втрати функціональності при off.

---

## Alternatives (розглянуто, відхилено)

1. **Лишити ADR-0077 polite-таймінг** для тулбару — відхилено owner-ом: непередбачувано, «забуду як працює»; і не узгоджено з рештою підказок.
2. **Окремі перемикачі per-поверхня** (тулбар / зони / структури окремо) — відхилено: owner хоче ОДИН перемикач, «згруповані в одному місці, вмикаються спільно».
3. **Overlay-підказки завжди-он** (як було) — відхилено: заважають, «дуже заважають мої підказки для зон»; трейдеру потрібен чистий графік.
4. **Прибрати overlay-підказки зовсім** — відхилено: частково несуть інформацію для навчання; правильно — сховати за перемикачем, не видалити.

---

## Consequences

**Позитив:** дефолтно чистий графік (нічого не відволікає); усі підказки новачка вмикаються/вимикаються з одного місця (☰ «Підказки»); передбачувана поведінка тулбару (2 стани); −90 LOC складного таймінгу; нуль втрати функціональності при off (це навчальні пояснення, не контроли).

**Межі:** перемикач глобальний (немає per-категорії гранульованості — навмисно, owner-вимога); title-підказки хрому вже були під цим стором (без змін).

**Live-verified** (Playwright, реальний ☰-клік): off → тиша (тулбар hover 800 мс — лейбл не з'явився; sweep 130 точок по зонах/структурах — `.smc-tooltip` не показано); реальний клік по ☰-перемикачу «Підказки» → localStorage `v4_show_hints=1` → on → і тулбар-лейбл, і overlay-tooltip з'являються. svelte-check 0 errors.

---

## Rollback

- Тулбар → git revert `DrawingToolbar` до ADR-0077 polite-таймінгу (повернути `revealCounts`/`dwell`/`reset` + prop `alwaysShowHints`).
- Overlay → прибрати `setHintsEnabled` guard у `OverlayRenderer.mousemove` + `$effect` у ChartPane (підказки знову завжди-он).
Кожне незалежне; стор `uiHints` не змінювався (лише охоплення).
