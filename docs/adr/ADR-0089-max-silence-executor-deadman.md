# ADR-0089: MAX_SILENCE = зовнішній executor dead-man (міряє published-пульс бота)

- **Status**: Accepted (owner GO 17-18.07.2026, серія Страйк Г; companion до
  trader-v3 ADR-099 — X31: тут ТІЛЬКИ платформа)
- **Date**: 2026-07-18
- **Deciders**: Стас (owner), Claude (R_PATCH_MASTER)
- **Related**: ADR-0048 (WakeEngine — «hours з created_at» семантика max_silence
  тут ПЕРЕОЗНАЧУЄТЬСЯ), trader-v3 ADR-099 (home-eval: session/silence Арчі
  евалюються ботом; сюди переїжджає лише dead-man страховка), форензика 17.07.

## Context

Вбивця #2 форензики 17.07 [VERIFIED wake_engine.py:96,235,393,490 +
wake_check.py:79-84]: MAX_SILENCE міряв elapsed від `self._last_wake_ts[symbol]`
— часу ОСТАННЬОЇ події, яку двигун сам опублікував. Кожен fired event ресетив
лічильник → «тиша» ніколи не росла (max проміжок XAU-подій за день = 2h) →
11.5 год реальної тиші Арчі невидимі. Порочне коло самовимірювання.

Після trader-v3 ADR-099 тиша АРЧІ міряється вдома (CHECK 1.5 бота). Але
домашній страж структурно сліпий до ОДНОГО кейсу: **завислий/мертвий
виконавець** — страж живе у тому самому процесі. Це єдина непокрита діра
класу «тиша», і її власник — зовнішній спостерігач = платформа.

## Alternatives

1. **Лишити як є** (self-measure): rail номінально живий, фактично мертвий
   (ніколи не True). Відхилено — це і є вбивця #2.
2. **Видалити платформний max_silence**: RETHINK §4 явно захищає dead-man
   rail; видалення = втрата останньої страховки на bot-hang. Відхилено.
3. **Пере-семантизація на published-пульс (обране)**: бот публікує
   `{ns}:archi:pulse` (HSET: `heartbeat` = «виконавець живий», кожен
   wake_sync; `market_voice_ts` — довідково для «Очей»). MAX_SILENCE міряє
   elapsed від `heartbeat` → чесний dead-man: спрацьовує ТІЛЬКИ коли
   виконавець реально мовчить (мертвий/завислий/Redis-відрізаний).

## Decision

- `WakeEngine`: кеш `_pulse_heartbeat_ms` (HGET `{ns}:archi:pulse heartbeat`,
  той самий 30s-каданс, що bot conditions; сек→ms). У condition-loop для
  `MAX_SILENCE` `last_wake_ts_ms` = пульс, коли він > 0; інакше legacy
  self-measure + one-shot `WAKE_PULSE_ABSENT` WARN (деградація гучна; кейс =
  бот ще без ADR-099-3). `core/smc/wake_check.py` НЕ чіпається (отримує інший
  timestamp). Drains structure/bar-close лишаються на engine-`last_wake` —
  вони про інше.
- `config.json`: `event_cooldown_s.max_silence` 0 → **3600** — коли dead-man
  істинний (бот мертвий), подія раз/год у чергу, не флуд щотіку 5 хв
  (LTRIM 100 обмежує чергу, але шум — теж борг).
- Поріг `max_silence_hours: 3.0` НЕ міняється — семантика тепер «3h без
  heartbeat виконавця» (каданс heartbeat = хвилини: кожен CHECK 0-синк +
  post_save_hook; крипто-події тримають його і у вихідні).

**Читач події (чесно)**: коли dead-man істинний, бот мертвий → LPUSH-подію
нікому дренувати. Реальний сигнал Стасу сьогодні = stale-індикатор «Очей»
(presence TTL) + supervisor. Прямий алерт платформа→Telegram повз бота —
окремий майбутній мікро-ADR (нова здатність платформи, секрети), не тут.

## Consequences

- **+** Вбивця #2 закритий і на платформному боці: rail нарешті міряє те, що
  обіцяв («грошова страховка» RETHINK §4 стає функціональною).
- **+** Нуль хибних спрацювань при живому боті (heartbeat свіжий щохвилини).
- **~** Пропущена умова у вікні «бот живий, Арчі тихий» більше НЕ платформна
  справа — вона домашня (ADR-099), розподіл власників істини завершений.
- **−** Читач події при справжньому dead-man — поки непрямий (Очі/supervisor);
  задокументовано вище.

## Rollback

`git revert` (пульс-гілка інертна без ключа: `WAKE_PULSE_ABSENT` → legacy
поведінка byte-identical). Config cooldown повернути 0.
