## Quality Axes

- **Ambition target**: **R2** — нова взаємодія-модель на наявному об'єкті (hover-афорданси + контекстні курсори), без нової backend-поверхні.
- **Maturity impact**: **M3 → M3.5** — об'єкт стає прямо-маніпульованим на hover (не лише після кліку); прибрано мертвий hover-grow код; курсор-система контекстна. Знижень немає.

## Контекст

Owner-вимога: інструменти малювання мають бути преміум, делікатні, ефективніші за TradingView. Ключове рішення — **дві розведені поверхні**: (1) на об'єкті — делікатне пряме маніпулювання наведенням (рух / кінці / видалити), БЕЗ налаштувань; (2) конфігурація (колір/товщина/стиль) — окремо, у лівому тулборі (ADR майбутньої фази). Ця ADR фіксує **поверхню 1**.

Метод — **не мокапи, а жива реальність**: обговорення словами → мала реальна зміна в живому чарті → live-verify (Playwright: реальний `page.mouse`, canvas `getImageData`, читання cursor) → тюнинг. Кожен крок вивірено проти живого рендера, не проти картинки.

---

## Рішення (поверхня 1 — на об'єкті)

### D1. Наведення проявляє афорданси; спокій = чистий об'єкт
У cursor-режимі hover на фігуру (або вибрану) рендерить: **порожні золоті кільця** на кінцях + **× по центру**. Пішов курсор — зникли. Без right-click-меню для цих дій (ADR-0078 меню лишається окремо, поки що).

### D2. Кінці — порожні кільця, центр прозорий
`renderHandles`: лише золоте кільце (stroke, `--toolbar-active-color`), БЕЗ заливки — видно свічку крізь центр. Стала (не росте на hover), щоб не затуляти точку прицілювання. **Наведену крапку НЕ малюємо** — там уже приціл (crosshair), кільце зайве.

### D3. Видалити — × по центру, делікатне
`renderDeleteButton` на центрі AABB фігури (`DELETE_BTN_RADIUS_PX` = клікабельна+реакційна зона). Без фону-кружка — лише хрестик з тонкою тінню; **червоніє при наведенні** (`--bear`). Клік → undoable `DELETE`. Keep-alive у `performHitTest`: над × рахуємо як hit тіла, щоб афорданси не зникали поки курсор тягнеться до нього.

### D4. Контекстні курсори (не «рука скрізь»)
Системна рука-глиф затуляла делікатні цілі. Тепер:
- **тіло** → `grab` (взяв-повів);
- **кінець** → кастомний **crosshair-приціл** (`CURSOR_RETICLE`, data-URI SVG): тонкий +, **відкритий центр** (видно точку на свічці), напівпрозорий, темний halo для читабельності, hotspot = центр;
- **×** → `default` (звичайна тонка стрілка — не ховає червоний ×).
`cursor:none` **відхилено** — губить орієнтацію.

### D5. Реакція трохи РАНІШЕ (~1мм)
Радіуси реакції більші за візуал, щоб «ловило» на підході, не «майже поклав»: `DELETE_BTN_RADIUS_PX = 13`, handle-hit = `hitTolerancePx + HANDLE_REACT_MARGIN_PX(4)`. Live-verified свіпом курсора (× reacts ≤13px, grab ≥15px).

### D6. Обсяг — один інструмент як шаблон
Модель спроектована на трендовій, тиражується: кільця через `renderHandles` (по `points`), × через AABB-центр — працює для hline/rect однаково. Магніт і курсор — окремі режими (курсор = цей select/edit-режим).

## Alternatives (розглянуто, відхилено)

1. **Плаваючий contextual-toolbar на об'єкті** (TradingView/Figma) — відхилено owner-ом: налаштування йдуть у тулбар, об'єкт лишається делікатним.
2. **`cursor:none` над точними цілями** — відхилено: губиться орієнтація («важко зорієнтуватись»).
3. **Залиті / зростаючі крапки** — відхилено: затуляють точку прицілювання. → порожні сталі кільця + приціл.
4. **Червоне halo навколо ×** — відхилено: тонкий `default`-курсор і так не ховає червоний ×, halo зайве.
5. **Мокап/концепт-артефакт наперед** — відхилено owner-ом (мокап ≠ реальність). → discuss → try live → tune.

## Consequences

**Позитив:** об'єкт лишається чистим; точне прицілювання прицілом з відкритим центром (краще за руку, що накриває); делікатно, преміум; реакція «ловить» вчасно; нуль backend-поверхні; client-only + undoable (ADR-0005).

**Межі / тривають тонкі поправки:**
- **Рука на тілі ще не прозора** (native `grab`) — outline/transparent = наступний крок.
- Mobile (`pointer:coarse`) — курсори desktop-first; touch long-press поки поза scope.
- Модель — baseline; owner очікує подальші тонкі поправки.

**Live-verified (реальний `page.mouse` / `getImageData` / cursor):** афорданси на hover (золото 88px), кінець тягнеться (t_ms+price зсув), × видаляє (→0), курсори (тіло=grab, кінець=reticle valid SVG, ×=default), порожні кільця (центр прозорий), наведена крапка схована (gold 0 vs 42), реакція ≤13px. svelte-check 0 errors, vitest tools 28/28.

## Rollback

Погранульно: курсори → повернути `updateCursor` до `pointer/grab`; афорданси → повернути `renderHandles` gate на `selectedId` + прибрати `renderDeleteButton`/`deleteBtn`; радіуси → 9 / `hitTolerancePx`. Кожне незалежне.
