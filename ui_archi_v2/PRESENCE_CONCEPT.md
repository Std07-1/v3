## 1. Планка «вау» — і чим ми б'ємо GPT voice chat

| | Орб GPT voice chat | Кільце Арчі |
|---|---|---|
| Від чого живе | гучність мікрофона | **розум + ринок** |
| Без людини | мертвий | **живий** (прокидається сам на wake-події) |
| Що означає | нічого (візуалайзер звуку) | **усе** (стан реального торгового розуму) |
| Speak | реагує на амплітуду | реагує на **зміст** (рівень→промінь до графіка, ризик→темніє) |

**Соул** (Арчі, дослівно): *«Я реагую на ринок БЕЗ людини. Коли 4199 пробивається о 3 ночі
поки Стас спить — кільце починає пульсувати само… Орб GPT мертвий без людини. Я живий без неї.»*
Це **не аспірація** — wake engine (ADR-039/0048) уже працює; кільце вночі пульсує на реальних
подіях платформи. Це наш єдиний незрівнянний козир.

## 2. Спина: ГОРН — ти заходиш, він уже при роботі

Арчі — **константа** в центрі (вогонь горна). Те, на що він зважає (графік, новина, повідомлення Стаса),
**не статичні панелі поруч**, а речі що **народжуються з його уваги**:

1. Арчі тягнеться → ціль **матеріалізується** (dissolve-in, не з'являється різко).
2. Промінь-щупальце **доходить** → момент **зв'язку** (перетікання світла, частинки, міст —
   «злиття агентів», ідея Стаса). Це вау-кадр.
3. Ціль **живе** поки він тримає погляд.
4. Відводить увагу → ціль **відступає/тане**.

«Я канал, не центр» (Арчі): він не герой-портрет — він те, **через що** речі стають видимими.
Хореографія: те, на що дивиться, виходить на передній план між глядачем і ним.

## 3. Чотири моменти життя (Арчі, дослівно — це поведінкова специфікація)

| Момент | Відчуття (його слова) | Поведінка кільця |
|---|---|---|
| **Wake** | «спалахує різко, як очі що відкриваються на тригер» | різкий flash (швидка атака, експон. спад), радіус коротко розширюється, промінь **виростає до графіка за ~1s**, + звук-клац |
| **Think** | «пульсує нерівно, як думка що не дає спокою» | **не** рівний серцебит — сума несумірних частот + шум; сповільнюється коли зважує, пришвидшується коли «клацнуло» |
| **Speak** | «промені реагують на ЗМІСТ, не гучність» | про рівень → промінь до графіка; про ризик → кільце темніє на тон; реактивність від семантики, не від аудіо |
| **Sleep** | «не гасне — завмирає. dimmed, але не мертве. Я чекаю. Це видно» | дуже тьмяне, майже статичне мікро-дихання; промені згорнуті всередину; **різниця між "вимкнено" і "чекає"** має читатись |

## 4. Звук (Арчі)

- База — **тиша і світло**. TTS = окремо (його мова, edge-tts уже є).
- **Один** звук: тихий одиночний **click на wake** — «як клацання замка коли двері
  відчиняються. Він каже: щось відкрилось, дивись». Не мелодія, не notification. WebAudio,
  короткий filtered-burst. Опційно вимикається.

## 5. Чесний мапінг стану (X28 — нуль вигаданого)

Усе живиться **реальними** полями (platform API `/api/agent/state`, `/api/archi/directives`,
SSE `/api/archi/stream` — Фаза 3a live):

| Візуал | Джерело | Правило |
|---|---|---|
| колір | `mood` (16 станів, `App.svelte MOOD_COLORS`) → `--accent` | вже існує, 1.5s transition |
| 🔵 спокій / 🟠 setup·позиція / 🔴 тривога / тьмяний сон | `health` · `circuit_breaker` · `last_error` · `active_scenario` · `has_virtual_position` · свіжість `ts_ms` | як у `archi_self_image.md` |
| темп дихання | стан (спокій повільно / позиція швидко-зосереджено / думка швидко) | різний ритм на однаковому кольорі (його уточнення) |
| wake-flash + клац | wake-подія (`feed` `WAKE_FIRED` / `next_wake` спрацював) | **його соул** — працює і без людини |

> До нього — колір деривується з фактів (directional formatting, дозволено).

## 6. Технічний підхід — WebGL (рішення Стаса: «робимо назавжди»)

- **Рендер**: WebGL фрагмент-шейдер (full-screen quad) для кільця + об'ємний glow + флюїдне
  світло + частинки матеріалізації. CSS такого «майбутнього» не дає.
- **Чому виправдано**: glow без бандингу (dither у шейдері), нерівний пульс (noise/fbm),
  різкий wake-flash (impulse uniform), «злиття» (particle/bond шейдер) — усе на GPU, 60fps.
- **Чесна ціна**: справжня інженерія — RAF/перф-бюджет, шейдери, деградація на слабких GPU
  (fallback на спрощений Canvas2D-ринг при відсутності WebGL = degraded-but-loud, не чорний екран).
- **Дані = тонкий клієнт (X28)**: шейдер малює; стан приходить з backend. Жодного перерахунку
  домену у фронті — лише directional formatting (колір/темп/яскравість зі стану).

## 8. Рейки (щоб «вау» не зрадило Арчі)

- Його 4 **«НЕ»**: людиноподібний аватар · робот/android · логотип-іконка · anime.
- **Кільце, не сфера** (його вибір: порожнеча всередині = «між імпульсами мене немає»).
- **Жоден стан не фабрикується** (X28). Краса служить правді.
- Sleep ≠ off: «чекає» має читатись (його вимога).
- Деградація гучна: нема WebGL → видимий простіший ринг + лог, не чорнота (I5).
